6. Key Features

  • Context-Aware: Settings are automatically scoped to your tool - no need to specify tool ID
  • Persistent: Settings are stored securely using electron-store and persist across app restarts
  • Isolated: Each tool has its own settings namespace - tools can't access each other's settings
  • Type-Safe: Supports any JSON-serializable value (strings, numbers, booleans, objects, arrays)
  • Simple API: Just four methods: getSettings(), getSetting(key), setSetting(key, value), setSettings(object)

7. Code Examples

Save a Setting

// Save a single setting
await toolboxAPI.settings.setSetting('theme', 'dark');
await toolboxAPI.settings.setSetting('autoRefresh', true);
await toolboxAPI.settings.setSetting('refreshInterval', 5000);

Get a Setting

// Get a single setting
const theme = await toolboxAPI.settings.getSetting('theme');
console.log('Current theme:', theme); // 'dark'

// Handle non-existent settings
const value = await toolboxAPI.settings.getSetting('nonExistent');
if (value === undefined) {
    console.log('Setting does not exist');
}

Get All Settings

// Get all settings for this tool
const allSettings = await toolboxAPI.settings.getSettings();
console.log('All settings:', allSettings);
// Returns: { theme: 'dark', autoRefresh: true, refreshInterval: 5000 }

Bulk Update

// Update multiple settings at once
await toolboxAPI.settings.setSettings({
    theme: 'light',
    autoRefresh: false,
    refreshInterval: 10000,
    lastSync: new Date().toISOString()
});

Complex Data Types

// Save complex objects and arrays
await toolboxAPI.settings.setSetting('userPreferences', {
    notifications: true,
    sound: false,
    columns: ['name', 'date', 'status']
});

await toolboxAPI.settings.setSetting('recentFiles', [
    '/path/to/file1.txt',
    '/path/to/file2.txt'
]);
